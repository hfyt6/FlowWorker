# 第六步：对话循环和流式处理

## 目标
实现完整的对话循环机制和高级流式响应处理，支持工具调用、上下文压缩和增量更新。

## 具体任务
- [ ] 6.1 实现核心对话循环 (runLoop)：
  - 用户输入处理和消息历史管理
  - API 请求构建（系统提示 + 历史 + 工具定义）
  - LLM Provider 调用封装
  - 流式响应处理
- [ ] 6.2 实现工具调用处理：
  - 工具调用识别和解析
  - 工具执行框架
  - 工具结果添加到消息历史
  - 递归对话循环（有工具调用时继续）
- [ ] 6.3 实现 SSE 流式响应处理：
  - 服务器端 SSE 实现
  - 客户端流解析器 (streamParser)
  - 增量内容更新
  - 工具调用事件处理
- [ ] 6.4 实现上下文压缩功能：
  - 手动压缩早期对话历史
  - 自动压缩策略
  - 压缩算法实现
- [ ] 6.5 实现对话控制功能：
  - 停止生成（中断流式响应）
  - 重新生成（请求 AI 重新生成回复）
  - 最大迭代次数限制
  - 错误阈值控制

## 预期成果
- 完整的对话循环机制
- 工具调用支持和处理
- 高效的 SSE 流式响应
- 上下文压缩能力
- 完善的对话控制功能

## 验收标准
- 对话循环能够正确处理用户输入和 AI 响应
- 工具调用能够被正确识别、执行和记录
- SSE 流式响应能够实时显示增量内容
- 上下文压缩能够有效减少 Token 使用
- 停止生成和重新生成功能正常工作
- 对话循环能够在适当条件下终止（无工具调用/达到最大迭代/用户取消/错误阈值）
