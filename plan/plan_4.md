# 第四步：记忆系统实现

## 目标
实现三层记忆系统架构，包括短期记忆（滑动窗口）、中期记忆（任务级）和长期记忆（跨任务），支持上下文管理和任务持久化。

## 具体任务
- [ ] 4.1 实现短期记忆（Context Memory）：
  - 滑动窗口管理器 (SlidingWindowManager)
  - Token 预算计算和自动修剪
  - 关键消息保留策略（系统提示、工具调用结果）
  - 上下文构建器 (ContextBuilder)
- [ ] 4.2 实现中期记忆（Task Memory）：
  - 任务状态实体 (Task)
  - 任务存储服务 (TaskStorage)
  - 工具调用记录 (ToolCallRecord)
  - 文件修改追踪 (FileModification)
  - 跨会话任务恢复机制
- [ ] 4.3 实现长期记忆（Long-term Memory）：
  - 自定义指令存储 (CustomInstructionStorage)
  - 模式偏好配置 (ModePreferences)
  - 全局配置记忆
  - 作用域管理（全局/工作区）
- [ ] 4.4 实现记忆服务集成：
  - MemoryService 统一接口
  - 与现有 Service 层集成
  - API 控制器扩展
- [ ] 4.5 实现前端记忆管理界面：
  - 任务记忆面板
  - 自定义指令编辑器
  - 记忆状态可视化

## 预期成果
- 完整的三层记忆系统架构
- 滑动窗口自动管理对话历史
- 任务状态持久化和恢复能力
- 自定义指令和模式偏好存储
- 记忆服务与现有系统无缝集成
- 前端记忆管理界面

## 验收标准
- 滑动窗口能够根据 Token 预算自动修剪历史
- 关键消息（系统提示、工具结果）能够优先保留
- 任务状态能够正确持久化到文件系统
- 能够从中断处恢复任务执行
- 自定义指令能够按作用域正确存储和应用
- 记忆系统不影响现有功能的正常运行
- 前端能够显示和管理记忆数据
